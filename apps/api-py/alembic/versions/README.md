# Migrations

No baseline revision is checked in yet — see plan §2.4. The `0001_baseline_stamp`
revision must be generated by running `alembic revision --autogenerate`
against a **staging clone of the actual production database** and verifying
the diff is empty (or fixing the models until it is) before it's trusted and
committed. That requires real database access this session did not have, so
it's left as the next concrete step rather than fabricated.

For local development against a fresh, empty Postgres, `alembic upgrade head`
is safe once a first revision exists (there's no live data to protect there).
`alembic stamp head` (never `upgrade head`) is the only safe operation
against any environment that already has the existing Node-created schema.
