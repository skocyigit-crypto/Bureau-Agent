# Overview

"Agent de Bureau" is a TypeScript-based pnpm monorepo for a French-language office agent application. Its core purpose is to centralize and manage phone calls, contacts, tasks, and messages for businesses, significantly boosting productivity through advanced AI integration and comprehensive analytics. The application features full-featured management modules, extensive AI capabilities for analysis, suggestions, and validation, and a multi-agent AI system. The vision is to deliver a premium, intelligent office management solution tailored for French-speaking markets, leveraging cutting-edge AI to automate and optimize daily administrative tasks.

# System Architecture

The project is a pnpm workspace monorepo using Node.js 24 and TypeScript 5.9.

## UI/UX Decisions
The application features a French UI with a deep navy and warm amber color scheme, built using `shadcn/ui` and `Recharts` for a modern, responsive experience. It also provides a premium device-aware experience with device environment detection, page transition animations, haptic feedback, safe-area CSS, and theme toggling (Dark/Light/System). The application is installable as a Progressive Web App (PWA) with offline capabilities.

## Technical Implementations
- **Backend:** Express 5, PostgreSQL, Drizzle ORM, Zod for validation, Orval for API codegen, and `esbuild` for CJS bundling.
- **Frontend:** React, Vite, Tailwind CSS, shadcn/ui, and Recharts.
- **AI Integration:** Leverages Gemini 2.5 Pro, OpenAI, and Anthropic for analytics, contextual suggestions, form validation, Q&A assistance, pattern recognition, email drafting, smart discovery, and an "Intelligence Centrale / Assistant Executif" for specialized tasks.
- **AI Cost & Usage Tracking:** Per-organisation token & cost tracking via `ai_usage` table. Records provider/model/route/tokens/cost/duration for every tracked AI call. Endpoint `GET /api/ai-usage/summary?days=N` returns aggregated totals, breakdowns by route/model/day, and recent errors. Error messages are sanitized (PII patterns redacted: email/phone/uuid/jwt/apikey/url) before storage.
- **Multi-Agent AI System:** Ten specialist agents cover various office roles, generating scored reports, orchestrated by a "Super Agent IA." Includes an "Oto-Pilot IA" for continuous platform monitoring and auto-correction.
- **Authentication & User Management:** Database-backed system with bcryptjs, PostgreSQL session store, cookie-based sessions, role-based access (Super Admin, Administrateur, Agent, Lecture seule), and account lockout. Features an admin-initiated email invitation system with cryptographic tokens and license limit enforcement. Password reset flow: `POST /api/auth/forgot-password` (sends time-limited token via email) + `POST /api/auth/reset-password` (validates token, updates hash). Login page has multi-mode UI (login / forgot / forgot_done / reset). Reset tokens stored as `reset_password_token` + `reset_password_expiry` on the `users` table.
- **Multi-Tenant Architecture:** `organisations` and `subscriptions` tables ensure tenant isolation with `organisation_id` foreign keys and `requireTenant` middleware. Super Admin interface for full CRUD on client organizations and license management.
- **Daily Digest (`GET /api/daily-digest` + `app/daily-digest.tsx`):** AI-powered daily work summary per user. Backend gathers today's user-scoped data in parallel (calls created/answered/missed, tasks created/completed/overdue, notes, calendar events, messages, audit actions, upcoming tasks & events for the week) then sends a structured prompt to Gemini 2.0 Flash which returns: `resume` (warm 2-3 sentence summary), `humeur` (positif/neutre/attention), `score` (0-100 productivity), `points_forts`, `suggestions` (3-5 items each typed as action/alerte/conseil/felicitation with priorite), `demain` (tomorrow's message + top 3 priorities). Mobile screen has: color-graded hero banner (green/blue/red based on humeur), score ring, 4-stat grid (calls/tasks/notes/events), detailed activity breakdown, points forts card, suggestion cards with color-coded borders, completed tasks list, upcoming tasks with due dates & priority dots, upcoming calendar events, and a "Pour demain" gradient card with numbered priorities. Entry point: More → RAPPORTS → "Mon Bilan du Jour". Route wired in `routes/index.ts` and modal stack in `_layout.tsx`.
- **Mobile App:** A full-featured companion Expo mobile app with complete feature parity for key modules. Includes a continuous privacy protection system (`contexts/PrivacyContext.tsx` + `components/PrivacyOverlay.tsx`): (1) **Background shield** — when app enters background/inactive state, content is immediately covered with a blurred logo overlay so app switcher never shows sensitive data; (2) **Auto-lock** — configurable inactivity timer (off/1/5/15/30 min) automatically locks the app; (3) **PIN lock** — 4-digit PIN with salted hash stored in AsyncStorage, shake animation on wrong attempt; (4) **Biometric unlock** — Face ID / Touch ID via `expo-local-authentication`, auto-triggered on foreground return; (5) **Sensitive data masking** — `MaskedText` component masks phone/email until tapped; (6) **Manual lock** — one-tap lock from Settings; (7) **Privacy settings card** in `settings.tsx` with toggles for all features and PIN management UI. `PrivacyProvider` wraps the entire app in `_layout.tsx`, `PrivacyOverlay` renders above all content as `zIndex: 9999`.

- **Security Hardening:** Global `requireAuth` and `requireTenant` middleware, Helmet, rate limiting, strict CORS, HPP protection, Zod validation, Drizzle ORM, structured error handling, mutex locks for AI processing, and advanced threat detection.
- **Guardian WAF (`middleware/guardian.ts`):** Outermost defense layer wired as the very first middleware in `app.ts`. Inspects 100% of incoming traffic before any other handler. 8 detection layers: (1) **Attack tool user-agents** — 35+ signatures (sqlmap, nikto, burpsuite, nmap, nuclei, gobuster, ffuf, wpscan, ...); (2) **Honeypot traps** — 50+ decoy paths (/.env, /wp-admin, /phpmyadmin, /.git/config, /backup.sql, ...); (3) **Suspicious URL patterns** — path traversal, PHP/ASP/JSP shells, backup files; (4) **JSON bomb protection** — max 25 nesting depth, max 5000 keys; (5) **HTTP method enforcement** — blocks TRACE, CONNECT, and other non-standard verbs; (6) **Oversized header detection** — blocks headers > 16KB; (7) **Empty User-Agent logging**; (8) **Behavioral profiling** — per-IP request rate, unique path diversity (>80 paths = scanner), error rate; escalating auto-ban (5min → 15min → 1h → 6h → 1day → permanent); loopback/private IPs are exempt. API endpoints: `GET /security/guardian/stats`, `/events`, `/banned`, `/profiles`; `DELETE /security/guardian/banned/:ip`. Real-time frontend panel in settings/security tab with auto-refresh (8s), tabbed view (events/bans/profiles), and unban button.
- **Tenant Guard Security Layer (`middleware/tenant-guard.ts`):** Comprehensive multi-tenant abuse prevention middleware covering: (1) **Tenant isolation** — `assertOrgOwnsUser` verifies every user CRUD operation is scoped to the caller's org; cross-tenant probes return 404 and are logged; (2) **Role escalation prevention** — `assertRoleAllowed` blocks tenant `administrateur` from assigning or creating `super_admin` accounts; (3) **Super-admin protection** — `assertTargetNotSuperAdmin` prevents any tenant user from modifying or deleting platform super-admin accounts; (4) **User quota enforcement** — `assertUserQuotaNotExceeded` checks `organisations.max_users` vs active user count before each creation, returns structured `USER_QUOTA_EXCEEDED` error; (5) **Privilege rank enforcement** — `assertCallerOutranks` prevents admins acting on peers/superiors; (6) **Input sanitisation** — `sanitiseUserPatch` strips unknown fields from user PATCH payloads; (7) **Rate limiting** — `checkSensitiveRateLimit` in-process per-user throttling for sensitive ops; (8) **Audit trail** — every security violation logs to `audit_logs` with IP, path, role, and org context. All guards integrated into `POST /auth/users`, `POST /auth/users/create-and-send`, `PATCH /auth/users/:id`, and `DELETE /auth/users/:id`.
